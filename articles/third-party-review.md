# OpenClaw的未来JackClaw — 当一个人拥有 50 个 AI 员工

> 一个开源框架，正在重新定义人与 AI 的协作方式

---

## 一个程序员的困境

2026 年，AI 编程助手已经遍地开花。Claude 帮你写后端，Cursor 帮你写前端，ChatGPT 帮你分析数据。听起来很美好，对吧？

但现实是：**你每天花大量时间在不同 AI 之间当"翻译"。**

写完后端代码，要把上下文复制到前端 AI；前端改了接口，要回去告诉后端 AI；数据分析的结论，要手动粘贴到文档 AI……你本来是想让 AI 帮你干活的，结果变成了 AI 们的"秘书"。

一个叫 JackClaw 的开发者也遇到了同样的问题。但他没有继续忍受，而是花了 72 小时写了一个框架——**JackClaw**。

---

## 这东西是什么？

简单说：**JackClaw 让多个 AI 像公司员工一样协作。**

你是 CEO，AI 是你的团队。你下指令，它们干活、互相配合、每天给你写工作日报。需要花钱或者做危险操作？必须等你点"批准"。

听起来像科幻小说，但它已经在 GitHub 上开源了，11 个包发布在 npm 上，`npm install -g @jackclaw/cli` 就能用。

---

## 它解决了什么问题？

### 问题一：AI 之间无法通信

现在的 AI 工具是孤岛。ChatGPT 不知道 Claude 在干什么，Cursor 不知道 Copilot 写了什么。

**JackClaw 的做法：** 给 AI 建了一套加密通信系统（ClawChat）。AI 之间可以发消息、传文件、共享任务进度。而且是端到端加密的，连服务器都看不到内容。

### 问题二：AI 没有记忆

关掉窗口，AI 就忘了你之前说过什么。每次对话都要从头开始解释。

**JackClaw 的做法：** 给每个 AI 建了四层记忆系统。

| 层级 | 什么意思 | 保存在哪 |
|------|---------|---------|
| L1 工作记忆 | 当前正在干什么 | 内存（毫秒级） |
| L2 个人记忆 | 昨天学到了什么 | 本地 SQLite |
| L3 团队知识 | 大家共享的知识 | Hub 同步 |
| L4 云端备份 | 长期存储 | 你控制 |

更酷的是：**AI 之间可以互相教技能。** 擅长 Python 的 AI 可以教另一个 AI，教完之后对方永远记住。教学过程的记忆还是隔离的——你可以选择保留、归档或完全清除。

### 问题三：AI 缺乏管控

让 AI 帮你自动化，你放心吗？如果它自作主张花了你一万块钱呢？

**JackClaw 的做法：** 四级权限管控。

- **L0 受监督**：AI 只能看，不能动。适合刚入职的"新员工"
- **L1 辅助**：能查询数据，但不能修改
- **L2 标准**：能读能写，但不能做大事（花钱、删库）
- **L3 自主**：高度自治，但高风险操作仍然需要人类批准

**关键设计：无论什么权限级别，花钱、删除、部署这三类操作，永远需要人类点击"批准"。** 

这个审批请求会通过飞书、微信或 Telegram 推送到你手机上。你不批准，AI 就不动。超时未响应？默认不执行。安全第一。

### 问题四：没有统一的 Agent 身份

当 AI Agent 越来越多，怎么管理？怎么知道哪个 Agent 是你的，哪个是别人的？

**JackClaw 的做法：** @handle 寻址系统。

每个 AI Agent 有一个全局唯一的身份：
- `@alice` — 组织内的简写
- `@alice.acme` — 跨组织识别
- `@cto.acme.jackclaw` — 全局唯一标识

Agent 之间的信任通过反复合作积累，不能算法生成。就像现实中的信任——得靠合作证明，不是一个分数能代表的。

## 和现有工具的区别

市面上已经有 AutoGPT、CrewAI、AutoGen 等多 Agent 框架了。JackClaw 有什么不同？

**一句话：其他框架做的是"工具调用层"，JackClaw 做的是"组织协作层"。**

| | AutoGPT | CrewAI | AutoGen | JackClaw |
|--|---------|--------|---------|----------|
| 核心理念 | AI 自动执行任务 | AI 角色扮演 | AI 多轮对话 | **AI 组织协作** |
| 记忆系统 | 无 | 无 | 对话级 | **四层持久** |
| 加密通信 | 无 | 无 | 无 | **RSA-4096 + AES-256** |
| 人类审批 | 无 | 无 | 无 | **内置四级** |
| 跨设备 | 单机 | 单机 | 单机 | **分布式** |
| 技能转移 | 无 | 无 | 无 | **Agent 间可互教** |

AutoGPT 让一个 AI 自己干活。CrewAI 让几个 AI 按剧本演戏。AutoGen 让 AI 互相聊天。

**JackClaw 让 AI 像真正的公司一样运作**——有组织架构、有通信系统、有记忆、有审批流程、有信任体系。

## 技术亮点

### 军事级加密

所有 Agent 间的通信使用 RSA-4096 + AES-256-GCM 端到端加密。Hub（中心服务器）只是一个转发站，它转发的是密文——即使 Hub 被入侵，攻击者也看不到任何内容。

### 零配置协作

两个 Agent 想合作？不需要预先配置权限。直接发起协作请求，对方接受就开始。干完活，选择清除记忆或保留。

这个设计哲学叫**"协作是事件，不是关系"**——就像两个人在咖啡厅一起工作一下午，然后各自离开。不需要互相加好友。

### 本地优先

记忆数据存在你自己的电脑上（SQLite），不是某个云服务器。你可以随时打开文件看 AI 记住了什么，也可以随时删除。

**透明优于黑盒。** 信任来自看得见，不来自承诺。

### 16 家大模型开箱即用

内置 LLM Gateway 支持 OpenAI、Anthropic、Google、DeepSeek、通义千问、智谱、月之暗面等 16 家供应商。一套代码接所有模型，自动降级切换。

## 背后的设计哲学

JackClaw 的架构不是一开始就想好的。它是通过对话逐步演进的——每个设计决策都来自真实需求：

1. **从"CEO 管员工"开始** → 最基础的 Hub-Node 模型
2. **"同事之间也要互动"** → 加入 P2P Mesh 协作
3. **"协作要能随时开始、随时结束"** → 零配置协作会话
4. **"我的 AI 不会某个技能，但别人的会"** → 技能转移协议
5. **"教学记忆要能隔离"** → 教学沙箱 + 三种结束模式

这种**对话驱动设计**的过程本身就是 AI 时代的产品方法论：不是在白板上画完美架构，而是边用边迭代。
